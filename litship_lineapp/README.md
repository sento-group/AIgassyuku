## 📊 機能詳細

### 1️⃣ メインダッシュボード (`pages/dashboard.html`)

**主な機能:**
- KPIカード（売上、スタッフ数、案件数、配置率）
- スタッフ管理（100名対応）
  - リアルタイム検索（名前・コード・駅）
  - スキルフィルター（MNP/光/クローザー/キャッチャー）
  - ページネーション（12件/ページ）
- 案件管理
  - ステータスフィルター（配置済み/未配置）
  - 配置進捗の可視化
- カレンダー表示
- 分析・レポート
  - スキル分布グラフ
  - 月別売上予測
  - TOPパフォーマンスランキング

**使い方:**
1. サイドバーから機能を選択
2. 検索バーでスタッフや案件を検索
3. フィルターで絞り込み
4. 詳細表示や配置操作

---

### 2️⃣ 稼働日提出 (`pages/availability-submit.html`)

**主な機能:**
- カレンダー形式で稼働可能日を選択
- 提出期限アラート（毎月15日）
- 提出状況の確認
- 過去データの閲覧

**使い方:**
1. カレンダーで稼働可能な日をクリック
2. 複数日選択可能
3. 「✅ 提出する」ボタンで確定

---

### 3️⃣ KPI管理 (`pages/kpi-dashboard.html`)

**主な機能:**
- **KPI概要タブ**: 今月の目標・実績・達成率
- **提出状況タブ**: スタッフ別の稼働日提出状況
- **売上予測タブ**: 来月の売上を自動計算
- **事業計画タブ**: 年間目標設定と月別進捗グラフ

**売上予測の仕組み:**
```
予測売上 = 平均単価 × 延べ稼働日数 × マッチング率
```

---

### 4️⃣ 実績管理 (`pages/performance.html`)

**主な機能:**
- 日々の実績報告入力
- 自動スキル更新（実績に基づく）
- スキルシート生成
- ワンタップ出力（PDF/Excel/画像/テキスト）

**実績報告の流れ:**
1. スタッフ・案件・実施日を選択
2. MNP、光、カード、端末の獲得数を入力
3. 評価（⭐1〜5）を選択
4. コメント入力（任意）
5. 「📝 実績を登録」で完了

## 💾 データ管理

### ローカルストレージ

すべてのデータは**ブラウザのローカルストレージ**に保存されます。

**保存されるデータ:**
- スタッフ情報（100名分）
- 案件情報
- 稼働可能日の提出データ
- 実績報告データ
- 事業計画データ

**バックアップ方法:**
ブラウザのデベロッパーツール（F12）→ Application → Local Storage で確認・エクスポート可能

## 🔧 カスタマイズ

### カラー変更

`assets/css/dashboard-v2.css` の `:root` セクションでカラー変数を変更

```css
:root {
    --navy-blue: #1A3A52;
    --turquoise-blue: #00A0B0;
    --light-blue: #E8F4F8;
}
```

### サイドバー幅変更

```css
:root {
    --sidebar-width: 260px;
}
```

### データの更新

`assets/data/data-100.js` ファイルを編集して、スタッフや案件を追加・変更できます。

## 🆘 トラブルシューティング

### データが表示されない
→ `assets/data/data-100.js` が正しく読み込まれているか確認

### スタイルが崩れる
→ ブラウザのキャッシュをクリア（Ctrl + F5）

### モバイルでメニューが表示されない
→ ハンバーガーアイコン（☰）をタップ

### リンクが機能しない
→ 相対パスが正しいか確認（pages/からassets/への参照は`../assets/`）
