## 🎯 対話的スライド作成プロンプトの解説

### なぜ「対話的」なのか？

**従来のスライド生成ツールの問題点：**
- 一度に全部生成してしまう → 修正が大変
- ユーザーの意図を完全に理解できない → 的外れな内容
- 構成を考える時間がない → 不満足な結果

**対話的なアプローチのメリット：**
- 段階的に進める → 修正が簡単
- ユーザーの意図を確認しながら進める → 精度が高い
- 構成から一緒に考える → 満足度が高い

### プロンプトの構造解説

#### 1. 役割の定義
```
あなたはスライド作成のプロフェッショナルです。
```
→ Geminiに明確な役割を与えることで、スライド作成に特化した回答を引き出す

#### 2. 基本情報の入力
```
■ スライドの基本情報
・テーマ：[ここに記入]
・対象者：[ここに記入] 
・目的：[ここに記入]
・スライド枚数：[ここに記入]
```
→ 最低限の情報でスライドの方向性を決める

#### 3. 対話的な作成手順
```
1. まず、スライドの構成案を提案してください
2. 私のフィードバックを受けて、構成を調整してください
3. 各スライドの内容を対話的に作成してください
4. 最終的に、Googleスライドにインポートできる形式で出力してください
```
→ 段階的に進めることで、ユーザーの意図を反映しながら完成させる

#### 4. 出力形式の指定
```
【出力してほしい形】
- スライド構成案（見出しのみ）
- 各スライドの詳細内容
- Googleスライド用のテキスト形式
```
→ Googleスライドにインポートしやすい形式を指定

### 対話の流れ（実例）

**ユーザー**：「営業提案のスライドを作りたいです」
- テーマ：営業提案
- 対象者：製造業のクライアント
- 目的：AI導入サービスの提案
- スライド枚数：10枚程度

**Gemini**：「スライド構成案を提案します：
1. タイトルスライド
2. 現状課題
3. 解決策のご提案
4. 期待効果
5. 導入スケジュール
6. まとめ
この構成で進めますか？修正があれば教えてください。」

**ユーザー**：「『競合比較』のスライドも追加してください」

**Gemini**：「承知しました。構成を更新します：
1. タイトルスライド
2. 現状課題
3. 解決策のご提案
4. 競合比較 ← 追加
5. 期待効果
6. 導入スケジュール
7. まとめ
それでは、各スライドの内容を作成していきます...」

このように、対話的に進めることで、ユーザーの意図を反映したスライドが完成します。
