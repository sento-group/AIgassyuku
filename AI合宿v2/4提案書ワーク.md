# 提案書たたき＆NotebookLMワーク

実習3（13:15-16:00）で使用する、企画アイデアを提案書のたたき台にしてNotebookLMで提案スライドを作成するワークの手順書です。

**ミニマムゴール**：提案書たたきが完成し、NotebookLMでスライド構成ができている

---

## 目的

- 頭の中にある企画アイデアを言語化する
- 提案書の構造を理解する
- 複数のAIツールを連携させる体験をする
- **AIサイクル（Input→加工→Output→Action）を実践で体感する**

---

## AIサイクルの流れ

```
【Input】企画アイデアのメモ（頭の中の情報）
    ↓
【加工】Cursorで構造化・たたき台作成
    ↓
【Output】提案書たたき → NotebookLMに渡す → 提案スライド完成
    ↓
【Action】関係者に共有、プレゼンに活用
```

**💡 ポイント**：
- Cursorで作った「提案書たたき」がOutput
- それをNotebookLMに渡すのがAction
- NotebookLMで「提案スライド」が生成される→新しいOutput
- **AIツールを連携させることで、より大きな成果が得られる**

---

## Step 1: 企画アイデアを書き出す（15分）

### 1.1 まず頭の中を吐き出す

新しいmdファイルを作成して、企画アイデアを書き出してください。
完璧じゃなくてOK。箇条書きでも、メモ書きでも大丈夫です。

**ファイル名例**：`企画アイデア-〇〇.md`

**書く内容**：
- 何をやりたいか
- なぜやりたいのか
- 誰のためにやるのか
- どんな効果があるか
- 思いついたことなんでも

### 1.2 書き出しに詰まったら

```
@wajima.mdc を参照しながら、企画アイデアの整理を手伝ってください。

私が考えている企画は〇〇です。
でも、うまく言語化できません。
質問をしながら整理を手伝ってください。
```

---

## Step 2: 提案書のたたき台を作成する（40分）

### 2.1 構造化を依頼

```
以下の企画アイデアを、提案書のたたき台として構造化してください：

【企画アイデア】
（Step1で書き出した内容をコピペ）

提案書の構成：
1. 背景・課題（なぜこの企画が必要か）
2. 提案内容（何をするか）
3. 期待効果（どんな成果が得られるか）
4. 実行計画（いつ・誰が・何をするか）
5. 必要リソース（お金・人・時間）
6. リスクと対策

各セクションについて、詳しく書いてください。
```

### 2.2 内容を深掘りする

```
「背景・課題」のセクションをもっと詳しくしてください。
データや具体例があると説得力が増します。
```

```
「期待効果」を数値で示せる部分はありますか？
定量的な効果と定性的な効果を分けて記載してください。
```

### 2.3 NotebookLM用に整形

```
この提案書たたきをNotebookLMに渡せる形式に整えてください。

要件：
- マークダウン形式
- 見出しと箇条書きで構造化
- 1つのファイルにまとめる
- 各セクションの要点が明確
```

---

## Step 3: NotebookLMで提案スライド化（30分）

### 3.1 NotebookLMを開く

1. ブラウザで [NotebookLM](https://notebooklm.google.com/) を開く
2. Googleアカウントでログイン
3. 「新しいノートブック」を作成

### 3.2 提案書たたきをソースとして追加

1. Cursorで作成した提案書たたきをコピー
2. NotebookLMで「ソースを追加」→「テキストをコピー&ペースト」
3. コピーした内容を貼り付け
4. 「保存」をクリック

### 3.3 提案スライドを生成

NotebookLMのチャット欄に以下を入力：

```
このソースをもとに、プレゼンテーション用のスライド構成を作成してください。

スライド構成：
- 表紙
- 目次
- 背景・課題（2-3スライド）
- 提案内容（2-3スライド）
- 期待効果（1-2スライド）
- 実行計画（1-2スライド）
- まとめ

各スライドに何を書くか、具体的に示してください。
```

### 3.4 さらに深掘り

```
「背景・課題」のスライドについて、もっとインパクトのある伝え方を提案してください。
```

```
このプレゼンを聞いた人が「やりたい！」と思うように、
期待効果の見せ方を工夫してください。
```

## Step 4: 仕上げ（20分）

### 4.1 最終確認チェックリスト

- [ ] 課題が明確に伝わるか
- [ ] 提案内容が具体的か
- [ ] 期待効果に説得力があるか
- [ ] 実行計画が現実的か
- [ ] 聞き手が「やりたい」と思えるか

### 4.2 発表準備

```
今日作成した提案書について、以下の形式で発表準備をしてください：

1. 【テーマ】何の提案か
2. 【AIサイクル】どうやって作成したか
   - Input：何を入れたか
   - 加工：Cursorで何をしたか
   - Output：何ができたか
   - Action：NotebookLMでどう活用したか
3. 【学び】この体験で気づいたこと

3分で話せる分量でまとめてください。
```

## 定着パックからテーマを選ぶ場合

企画アイデアが思いつかない場合は、[3定着パック.md](./3定着パック.md) の中からテーマを選んで取り組むこともできます。

**例**：
- Day 4: 企画アイデアのメモ→実行計画に落とし込む
- Day 8: 複数の議事録→共通の論点を抽出
- Day 13: 自分の戦略メモ3つ→矛盾点や優先順位を整理

## 困ったら

### 「企画アイデアが思いつかない」

```
@wajima.mdc を参照しながら、企画アイデアを一緒に考えてください。

私の会社は〇〇業で、今の課題は△△です。
この課題を解決するアイデアを3つ提案してください。
```

### 「提案書の構造がよくわからない」

```
提案書の書き方を教えてください。
「背景・課題」には具体的に何を書けばいいですか？
例を示してください。
```

### 「NotebookLMの使い方がわからない」

バディに聞いてください！
または、以下のプロンプトをCursorで試してください：

```
NotebookLMの基本的な使い方を教えてください。
ソースの追加方法と、チャットでの指示の仕方を説明してください。
```

## 💡 wajima.mdcを活用する

進め方に迷ったら、[@wajima.mdc](./../.cursor/rules/wajima.mdc) を参照しながら対話してみてください。

```
@wajima.mdc を参照しながら、提案書作成の進め方についてアドバイスをください。
今、〇〇で詰まっています。
```

> **なぜwajima.mdcを勧めるのか**：
> 初心者がよくやりがちなのは「Inputがないのに対話ばかりして進まない」ことです。
> wajima.mdcは「まずInputを用意する」「小さく始める」といった考え方をガイドしてくれます。

## AIサイクルの振り返り

この実習で体験したAIサイクル：

```
【Input】頭の中の企画アイデア
    ↓
【加工】Cursorで構造化（提案書の型に当てはめる）
    ↓
【Output】提案書のたたき台
    ↓
【Action】NotebookLMに渡す
    ↓
【新しいOutput】提案スライド
```

**💡 学びのポイント**：
1. 頭の中の情報（暗黙知）を言語化することで、再利用可能な資産になる
2. AIツールを連携させることで、より大きな成果が得られる
3. 「たたき台を作る→他のツールに渡す」がAIサイクルのAction
