# Git 同期 (@sync)

「@sync」でリモートと自動同期。**エディタ・ページャは一切使わない**（`--no-edit` / `--no-pager` または `| cat` で完結）。

## 手順
1. `git status` → 未コミットがあれば `git commit -m "🤖 自動コミット: <概要>"`
2. `git fetch origin` → 取得予定の変更を一覧で通知（名前変更・移動・追加・削除）
3. `git pull origin main --no-edit --no-rebase`（失敗時は `git merge origin/main --no-edit`）
4. **コンフリクト時**: マーカーを読み取り両方維持する案を生成 → 差分をチャットで提示 → 承認後に適用して `git add .` → `git commit -m "🤖 自動コミット: コンフリクトを解決"`
5. `git push origin main`
6. 取り込まれた変更をサマリーで通知

## 必須
- コミットは必ず `-m` でメッセージ指定
- diff 等は `--no-pager` または `| cat`
- マージ失敗・解けないコンフリクトは中断してユーザーに確認

## 応答例
- 開始: `🔄 Git同期を開始します`
- 完了: `✅ 同期完了！ ローカル: X件 / リモート: Y件更新`
- コンフリクト: 解決案の diff を出し「この解決案でよろしいですか？」

## エラー時
認証→確認 / ネット→接続確認 / 解けないコンフリクト→処理中断
