# マッチングシステム

ビジネス合宿での人のマッチングを効率的に行うためのシステムです。

---

## 📁 フォルダ構成

```
マッチング/
├── README.md（このファイル）
├── マッチング基準.mdc（マッチング処理ロジック）
├── 参加者データ/
│   ├── 参加者リスト.md（全参加者の基本情報）
│   ├── スキル・興味マップ.md（スキルと興味の整理）
│   └── 個別/
│       ├── 田中太郎.md
│       ├── 佐藤花子.md
│       ├── 鈴木一郎.md
│       ├── 山田美咲.md
│       └── 高橋健.md
└── マッチング結果/
    └── サンプル-マッチング結果.md（マッチング結果のサンプル）
```

---

## 🎯 このシステムでできること

1. **参加者同士の相性を判断**
   - スキルの相補性
   - 興味・関心の一致度
   - 目標の方向性
   - 性格・コミュニケーションスタイル

2. **最適なペア・チームを提案**
   - 1対1のマッチング
   - 3-4人のチーム編成
   - スコアリングによる優先順位づけ

3. **会話のきっかけを提供**
   - マッチング理由の明確化
   - 具体的な会話テーマの提案

---

## 🚀 使い方

### ステップ1：参加者情報の入力
1. `参加者データ/参加者リスト.md` に新しい参加者を追加
2. `参加者データ/個別/` に詳細ファイルを作成（既存ファイルをコピーして編集）

### ステップ2：マッチング実行
Cursorで以下のように指示：

```
@マッチング基準.mdc @参加者リスト.md
参加者リストを見て、相性の良いペアを提案してください
```

または：

```
@マッチング基準.mdc @田中太郎.md @参加者リスト.md
田中太郎さんに合う人を3人ピックアップして、理由も教えてください
```

### ステップ3：結果の保存
マッチング結果を `マッチング結果/` フォルダに保存：
- ファイル名例：`2025-11-03-マッチング結果.md`

---

## 📊 マッチングの判断軸

### 1. スキル相性（30%）
- 異なるスキルを持つ人同士（相補性重視）
- 「教える人」と「学びたい人」のマッチング

### 2. 興味・関心の一致（40%）
- 同じテーマ・課題に興味がある人同士（共通性重視）

### 3. 目標の方向性（20%）
- 達成したいことが似ている人同士

### 4. 性格・コミュニケーションスタイル（10%）
- バランスを考慮（アウトプット型 × インプット型など）

## 💡 実践例

### 例1：特定の人に合う相手を探す
```
@マッチング基準.mdc @佐藤花子.md @参加者リスト.md
佐藤花子さんに合う人を提案して、会話のきっかけも教えてください
```

### 例2：全体のマッチングを実行
```
@マッチング基準.mdc @参加者リスト.md @スキル・興味マップ.md
全参加者のマッチングを実行して、強く推奨するペアをトップ5で教えてください
```

### 例3：チーム編成を依頼
```
@マッチング基準.mdc @参加者リスト.md
3-4人のチームを3つ作ってください。それぞれのチームの強みと適したテーマも教えてください
```

## 🔧 カスタマイズ方法

### マッチング基準の調整
`マッチング基準.mdc` の比重を変更：
- **スキル重視の場合**：スキル相性を50%に
- **交流重視の場合**：興味一致を60%に
- **チームビルディング重視の場合**：性格バランスを30%に

### 新しい判断軸の追加
`マッチング基準.mdc` に新しい判断軸を追加：
- 業界の近さ
- 地域の近さ
- 年齢・キャリアの近さ
